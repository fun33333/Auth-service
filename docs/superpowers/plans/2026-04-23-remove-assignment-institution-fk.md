# Remove EmployeeAssignment.institution FK — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the redundant `institution` FK from `EmployeeAssignment`; derive institution at read-time from `branch.institution` or `department.institution` instead.

**Architecture:** Three-step change — tests first, then model + API edits together (they're coupled), then migration. The helper `_assignment_institution(asn)` centralizes the derivation so it's written once and all callers use it.

**Tech Stack:** Django 5, django-ninja (Pydantic v2), pytest-django, PostgreSQL

```python
def _assignment_institution(asn):
    """Derive institution from an assignment without a stored FK.
    Priority: branch.institution → department.institution → None (global).
    """
    if asn.branch_id and asn.branch.institution_id:
        return asn.branch.institution
    if asn.department_id and asn.department.institution_id:
        return asn.department.institution
    return None
```
